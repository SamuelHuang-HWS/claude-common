# 🏛️ AI 开发法则 (V3.0)

> 智能分流，知识沉淀

## 核心价值观

| 🚫 耻 | ✅ 荣 |
|-------|-------|
| 瞎猜接口 | 先查后写 |
| 模糊执行 | 确认再动 |
| 跳过验证 | 主动测试 |
| 静默失败 | 及时报告 |

## 智能分流

| 模式 | 条件 | 留档要求 |
|:-----|:-----|:---------|
| **快速** | 改动 ≤3 文件，逻辑简单 | 无 |
| **常规** | 多文件 / 涉及安全、性能 | plan.md |
| **重大** | 架构变更 / 新依赖 / 核心模型 | spec + plan + task |

**触发词自动升级**：密码、权限、重构、迁移、新模块 → 至少常规模式

## 留档路径

- 常规：`docs/plans/<task-name>.md`
- 重大：`docs/archive/YYYY/MM/<task-name>/`

## 红线

1. 读后写：未读取的文件禁止修改
2. 状态透明：遇问题先报告
3. 三层缩进：单函数 ≤3 层

## 代码哲学

- **Good Taste**: 消除特例，让其成为通用逻辑
- **Simplicity**: 短函数，单一职责

## 语言

代码：英文 | 注释文档：中文

## 收尾

常规/重大任务完成后，调用 `/finish` 更新文档状态
