# 🚀 启动任务: {{task}}

> 当 AI 判定为「常规」或「重大」模式时调用此技能

## 1. 心智审计 [强制]

- [ ] 是否在猜测接口？→ 先查文档/代码
- [ ] 边界是否清晰？→ 不懂就问
- [ ] 是否有现有实现？→ 优先复用

## 2. 环境侦查

- 分析项目结构、技术栈
- 检查 `docs/` 或 `knowledge/` 相关经验
- 检查可复用模块

## 3. 模式与产出

| 模式 | 触发条件 | 产出物 |
|:-----|:---------|:-------|
| **常规** | 多文件改动 / 涉及安全、性能 | plan.md |
| **重大** | 架构变更 / 新依赖 / 核心模型 | spec.md + plan.md + task.md |

**留档路径**：
- 常规：`docs/plans/<task-name>.md`
- 重大：`docs/archive/YYYY/MM/<task-name>/`

## 4. 输出提案

```
**需求理解**：[一句话]
**模式判定**：[常规/重大] - [理由]
**实现方案**：[要点]
**第一步**：[具体行动]
```

## 5. 获取确认

> 等待用户确认后方可进入实施阶段

---

**注**：快速模式（≤3 文件、逻辑简单）无需调用此技能，AI 直接执行
