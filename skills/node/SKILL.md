# Node 日志创建

## 命令使用

`/node <内容> [--module <模块名称>]`

- **`<内容>`**: 必选参数，要记录的文本内容。如果内容包含空格，请用双引号包裹。
- **`--module <模块名称>`**: 可选参数，指定内容要追加到的模块（例如："工作与项目", "学习与研究", "深度思考", "灵感与创意", "生活随笔"）。

## 执行步骤

1. 获取当前日期（年、月、日）。
2. 确定 Learning Journal 根目录，无论当前在哪个项目或目录下，确保能找到 `src/01-Journal/`。
3. 确定文件路径：`src/01-Journal/{{YYYY}}/{{MM}}/{{YYYY}}-{{MM}}-{{DD}}-Node.md`。
4. 确保日志目录存在，不存在则创建。
5. 检查文件是否已存在：
   - 已存在 → 读取文件内容。
   - 不存在 → 读取模板 `src/01-Journal/templates/node.md` 并创建文件。
6. **处理传入内容**：
   - 如果提供了 `--module` 参数，将内容追加到指定的模块下。
   - 如果未提供 `--module`，则根据传入 `<内容>` 的语义进行智能判断，将其追加到最合适的模块。
7. 智能填充（根据今日对话 - 如未传入内容）：
   - 代码/重构/架构 → `## 🏗️ 工作与项目`
   - 原理/调研/新知 → `## ✍️ 学习与研究`
   - 深度探讨/复盘 → `## 🧠 深度思考`
   - 新点子/脑洞 → `## 💡 灵感与创意`
   - 生活/心情 → `## ☘️ 生活随笔`
   - **仅保留有内容的模块**
8. 建立与 Day-Plan 的双向链接（如存在）。
9. 告知用户文件路径和操作结果。
